# Plan: Add SUMMARY.md auto-generation workflow

## Goal
将 `src/SUMMARY.md` 加入 `.gitignore`，并通过 GitHub Actions workflow 在每次 push 时自动生成。

## Changes

### 1. `.gitignore` — 添加 SUMMARY.md
```
src/SUMMARY.md
```

### 2. `.github/workflows/generate-summary.yml` — 新建 workflow
- 触发条件: push 到 main 分支
- 步骤:
  1. checkout
  2. 运行脚本扫描 `src/A-Z/` 目录下所有 `.md` 文件（排除 `README.md`），按字母分组生成 `src/SUMMARY.md`
  3. 如果有变更则 commit + push 回 main

### 3. 生成逻辑
- 遍历 `src/` 下 A-Z 26 个字母目录
- 每个目录下找到所有 `*.md`（排除 `README.md`）
- 按文件名排序，生成格式:
  ```markdown
  - [字母](./字母/README.md)
    - [username](./字母/username.md)
  ```
- 空目录只输出 `- [字母](./字母/README.md)`

## Verification
- 本地运行生成脚本验证输出与当前 SUMMARY.md 一致
- push 后检查 Actions 是否成功执行
